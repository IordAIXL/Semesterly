## What the new research adds

### 1. Market direction

The research reinforces that productivity/wellness/calendar apps are moving toward:

- AI-assisted planning
- Lower-friction workflows
- Habit and wellness integration
- Unified dashboards instead of isolated tools
- Personalized daily summaries

This supports Semesterly's original positioning: not a generic calendar, but a daily decision layer.

### 2. Target sentiment

For Gen Z / college students, the strongest product angle is:

- reduce cognitive load
- avoid manual busywork
- make planning feel automatic
- help mental wellness by making the day feel less chaotic
- stay affordable

The app should not feel like corporate productivity software.

### 3. Competitive lesson

Microsoft/Outlook, Google Calendar, and Apple Calendar are not clean direct financial comps because their calendar products are bundled into larger ecosystems. That means Semesterly should not try to beat them as a standalone calendar.

The smarter strategy is to sit above/alongside those tools as a student-specific command center.

### 4. Business model

The research points toward a fully useful freemium model plus a very affordable premium tier.

Working pricing assumption:

- Free: core courses, assignments, events, dashboard, basic prioritization
- Premium: around `$10/year`
- Premium should unlock convenience and advanced intelligence, not basic usability

Possible premium features:

- advanced AI prioritization
- syllabus parsing
- deeper calendar sync
- dashboard customization
- analytics/insights
- smarter recurring workflows

### 5. Funding/financial assumptions

The financial report suggests:

- avoid heavy debt
- keep spending disciplined
- invest first in product/engineering and user retention
- integrations and AI are worthwhile only if they improve retention or subscription value
- hypothetical 18-month funding need: `$400,000`
- suggested structure: 80% equity / 20% debt

For the current build, this translates to: build the useful MVP before expensive integrations.

## Product implications

### Keep

- Student-first daily dashboard
- Course/assignment/event organization
- Automatic priority list
- Morning summary
- Freemium accessibility

### Add to roadmap

- Sentiment monitoring / feedback loop after beta
- Explicit mental-load reduction language in marketing
- Wellness/habit integrations as later differentiators
- Premium tier around affordability, not paywalling the core

### Be careful with

- Overpromising autonomous AI too early
- Building wellness integrations before academic workflow works
- Treating AI-generated market claims as verified fact
- Competing head-on with Google/Microsoft/Apple calendars

## Current build decision

The MVP still starts manual-first:

1. courses
2. assignments
3. events
4. dashboard
5. priority engine
6. onboarding
7. calendar import
8. AI summaries/prioritization
9. wellness/habit integrations

Reason: students need clarity first. Integrations make it faster later, but they do not replace the core product experience.
