# Semesterly Roadmap

## Phase 0 — Definition

Status: started.

Deliverables:

- Product brief
- Architecture
- UX plan
- MVP scope
- Starter project structure

## Phase 1 — Prototype

Goal: prove the core daily dashboard.

Build:

- Mock data
- Priority scoring function
- Dashboard UI
- Course/task/event models in TypeScript
- Static week view

Success:

- We can demo the product story without a database.

## Phase 2 — Local MVP

Goal: real data entry and persistence.

Build:

- Prisma schema
- SQLite database
- CRUD for courses
- CRUD for tasks
- CRUD for events
- Dashboard from DB

Success:

- One student can manually run their week in Semesterly.

## Phase 3 — Beta MVP

Goal: usable by first testers.

Build:

- Auth
- Production DB
- Deploy
- Mobile polish
- Onboarding/setup flow
- Basic analytics

Success:

- 10–25 real students can use it without handholding.

## Phase 4 — Differentiators

Goal: become more than a manual planner.

Build:

- Calendar import
- AI daily summary
- Syllabus upload/parser
- Smarter priority learning

Success:

- Users say Semesterly saves them planning time.

## Phase 5 — Campus launch

Goal: A&M focused launch.

Build:

- Referral loop
- Campus-specific landing page
- Feedback channel
- Ambassador/content playbook

Success:

- Retention and word-of-mouth justify broader rollout.
