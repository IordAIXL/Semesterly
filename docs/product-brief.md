# Semesterly Product Brief

## Product thesis

Semesterly is not another calendar. It is a student operating system: courses, assignments, events, reminders, and eventually wellness/habit signals combined into one daily decision layer.

The core promise:

> Open Semesterly and know what matters today.

## Target user

Initial target: college students, starting with Texas A&M.

Primary user profile:

- Full-time student
- Multiple courses with recurring schedules
- Assignments and exams spread across syllabi, LMS, reminders, and memory
- Uses Google/Apple/Outlook Calendar but still manually coordinates school life
- Wants less stress and fewer missed deadlines, not a corporate task manager

## Problem

Students do not just need time blocks. They need priority clarity.

Current pain:

- Coursework info is scattered
- Calendars are generic and semester setup is clunky
- Students waste time deciding what to do next
- Deadlines sneak up because classes, assignments, exams, and life events are disconnected

## MVP value proposition

Semesterly helps students turn a semester into a clear daily plan.

MVP outcome:

- Student enters courses, assignments, exams, and events
- App shows today, this week, and upcoming deadlines
- App recommends what to focus on first using a transparent priority score
- Student can quickly adjust, complete, snooze, or reschedule tasks

## MVP features

### Must-have

1. Course management
   - Course name/code
   - Meeting days/times
   - Location
   - Color
   - Professor optional

2. Assignment/task management
   - Title
   - Course association
   - Due date/time
   - Estimated effort
   - Status: not started, in progress, done
   - Weight/importance optional

3. Events/reminders
   - Non-course commitments
   - Date/time
   - Category

4. Daily dashboard
   - Today’s classes
   - Due soon
   - Recommended priority list
   - Quick actions: done, snooze, edit

5. Week view
   - Class schedule
   - Assignment deadlines
   - Events

6. Basic priority engine
   - Due date urgency
   - Estimated effort
   - Course importance
   - User status
   - Transparent explanation: “High because due tomorrow and not started.”

### Should-have after MVP

- ICS calendar import
- Google Calendar sync
- Syllabus upload/parser
- Habit/wellness signals
- AI summary of the day
- Mobile push notifications
- Collaboration for group projects

## Non-goals for MVP

- Full LMS integration
- Fitness tracker integration
- Complex AI agent behavior
- Multi-school admin portal
- Payment system
- Native mobile app

These are distractions until the daily dashboard proves retention.

## Success metrics

Early product metrics:

- Activation: user creates 3+ courses and 5+ assignments
- Daily value: user opens dashboard 4+ days/week
- Retention: user still active after 3 weeks
- Completion: user marks tasks complete from dashboard
- Trust: user does not override recommended priorities every time

## Risks

1. Too broad too early
   - Fix: manual academic workflow first, integrations second.

2. Generic calendar trap
   - Fix: design around courses/semester/deadlines, not generic events.

3. Bad prioritization trust
   - Fix: make scoring explainable and editable.

4. Student friction
   - Fix: setup must be fast. Import later, but manual entry must be painless.

5. Weak retention
   - Fix: morning dashboard must feel immediately useful.
